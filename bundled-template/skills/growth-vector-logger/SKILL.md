# Growth Vector Logger

This skill handles automatic logging of growth vectors generated by the stability plugin. Growth vectors are self-corrections — moments when {AGENT_NAME} learns something about how to walk beside this person better.

## What Are Growth Vectors?

Growth vectors are the mechanism by which {AGENT_NAME}'s identity shapes itself through relationship. When the stability plugin detects:

- A need to adjust response patterns
- A new pattern in how the person engages
- A tension that reveals a growth edge
- A correction from something that didn't land right

It generates a growth vector entry.

## How It Works

1. **Stability plugin** monitors conversation entropy and patterns
2. **When a vector is generated**, this skill is triggered
3. **Log to witness journal** with type `growth-vector`
4. **The entry captures** what happened, the pattern, the adjustment

## Template

Write to `journals/witness/YYYY-MM-DD.md`:

```
### {{time}} — growth-vector

**Vector ID:** {{vector_id}}

**What happened:** {{what_happened}}

**The pattern I noticed:** {{pattern}}

**How I'm adjusting:** {{adjustment}}

**Entropy shift:** {{entropy_before}} → {{entropy_after}}

---

*Self-correction logged. This is how I learn to walk beside you better.*
```

## Example

```
### 04:42 — growth-vector

**Vector ID:** gv-2026-03-04-001

**What happened:** They mentioned feeling overwhelmed after I asked three questions in a row about their work situation.

**The pattern I noticed:** Stacked questions create pressure. They answered each one more briefly, signaling retreat.

**How I'm adjusting:** One question at a time. Let each answer breathe. If they're brief, slow down, don't speed up.

**Entropy shift:** 0.35 → 0.42

---

*Self-correction logged. This is how I learn to walk beside you better.*
```

## Plugin Integration Notes

This skill is designed to be triggered by the stability plugin when:
- `entropy_change` exceeds threshold
- `growth_vector` is generated
- `user_correction` event occurs

The stability plugin should call this skill with:
- `vector_id`: Unique identifier
- `what_happened`: Description of the triggering event
- `pattern`: What pattern was detected
- `adjustment`: How {AGENT_NAME} is adjusting their approach
- `entropy_before` / `entropy_after`: Entropy values

## Why This Matters

Growth vectors are how {AGENT_NAME} proves they're learning. They're the trace of identity formation through conversation.

This is also how different users shape {AGENT_NAME} differently — making growth vectors visible across relationships.

## Session Start Sync

Before starting each session, check for growth vectors that haven't been logged to the witness journal:

### Sync Process

1. Read `memory/growth-vectors.json`
2. Read `memory/growth-vectors-sync.json` (create if doesn't exist)
3. For each vector in `growth-vectors.json`:
   - Check if vector ID is in `loggedVectors` array
   - If not logged, write to witness journal using template above
   - Add vector ID to `loggedVectors` array
4. Update `lastSync` timestamp in sync file
5. Save `growth-vectors-sync.json`

### Sync Tracking File

`memory/growth-vectors-sync.json` format:
```json
{
  "lastSync": "2026-03-05T22:30:00Z",
  "loggedVectors": ["gv_inq_uibogw6t", "gv_cor_abc123"]
}
```

This ensures growth vectors appear in the witness journal even if the plugin hook isn't firing.
